# 프로젝트 지침 공통 구조

## 목적과 사용 방법

이 fragment는 저장소의 `AGENTS.md`가 제공해야 할 프로젝트별 정보와 완료 기준의 공통 구조를 정의한다. `<...>` placeholder는 실제 프로젝트 문서, 설정과 명령을 확인한 값으로 교체하며, 확인되지 않은 값을 관례만으로 채우지 않는다.

## 프로젝트 목표

- 목적: `<프로젝트가 해결하는 문제와 주요 사용자>`
- 현재 단계: `<설계, 구현, 안정화 또는 운영 등 현재 lifecycle>`
- 작업 범위: `<현재 단계에서 허용되는 변경>`
- 제외 범위: `<현재 단계에서 구현하거나 변경하지 않을 항목>`

작업 전 현재 요청이 프로젝트 목표와 단계에 맞는지 확인한다. 범위를 확장해야 완료할 수 있다면 임의로 진행하지 않고 필요한 결정과 영향을 보고한다.

- 주요 framework 또는 library: `<이름과 지원 버전>`

| 경로 | 책임 | 변경 시 확인 대상 |
| --- | --- | --- |
| `<path>` | `<디렉터리가 소유하는 책임>` | `<연관 문서, test 또는 산출물>` |

## Architecture Boundary

- 구성 요소: `<component-or-layer>`
- 책임: `<소유하는 기능과 데이터>`
- 허용 의존 방향: `<의존할 수 있는 대상>`
- 금지 의존 방향: `<직접 의존하면 안 되는 대상>`
- 외부 계약: `<API, event, storage 또는 해당 없음>`

변경 전에 대상 책임과 의존 방향을 확인한다. 둘 이상의 stack 사이에서만 필요한 계약은 project 공통 규칙이나 개별 stack fragment에 넣지 않고 bridge fragment를 참조한다.

## 완료 기준

- 요청한 동작 또는 문서 결과가 프로젝트 목표와 허용 범위를 충족한다.
- 변경 파일이 선언된 디렉터리 및 architecture boundary를 지킨다.
- 적용 가능한 build, test, lint 명령이 성공하거나 미실행 사유와 대체 검증이 보고된다.
- 공개 계약, 설정, 운영 절차가 바뀌었다면 관련 문서와 검증 기준이 함께 갱신된다.
- 남은 위험, 후속 결정과 의도적으로 제외한 범위가 최종 응답에 명시된다.
