# architecture

## 목적

배포 단위와 내부 의존성 방향에 관한 architecture 선택지를 독립된 책임으로 관리합니다.

## 포함된 문서 자산

- architecture 유형별 `agents.fragment.md`
- 함께 선택 가능한 유형과 충돌 조건에 관한 메타데이터

## 현재 범위

Monolith, Modular Monolith, MSA, Clean, Hexagonal과 CLI Automation fragment가 존재하며 `registry/fragments.yml`에서 조회할 수 있습니다. Architecture template과 예제 코드는 포함하지 않습니다.
