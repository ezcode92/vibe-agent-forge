# <Skill 표시 이름>

> 이 파일은 Codex, Claude Code, Gemini CLI에 공통으로 적용할 수 있는 에이전트 중립 skill 문서의 템플릿입니다. 실제 에이전트별 파일 배치와 기능 차이는 adapter가 담당합니다.

## Skill 목적

<이 skill이 해결하는 하나의 작업 목적과 책임지지 않는 범위를 설명합니다. Skill은 사용자나 상위 지침이 부여하지 않은 권한을 추가하지 않습니다.>

## 사용 조건

- 사용 시점: <명시 호출 또는 작업 상황 등 적용 조건>
- 선행 조건: <필요한 상태, 도구 또는 확인 사항>
- 사용하지 않는 조건: <다른 skill이나 일반 지침으로 처리해야 하는 경우>

## 입력 Context

- 필수 입력: <작업을 시작하는 데 필요한 파일, 요청 또는 상태 정보>
- 선택 입력: <있을 때 판단 정확도를 높이는 정보>
- 제외 대상: <secret, 무관한 파일 또는 읽지 않아야 할 범위>

필요한 context만 좁혀서 읽고, 확인되지 않은 내용은 사실로 가정하지 않습니다.

## 작업 절차

1. <현재 상태와 작업 범위를 확인합니다.>
2. <필요한 증거를 수집하고 판단 기준을 적용합니다.>
3. <허용된 범위에서 작업을 수행합니다.>
4. <가장 좁은 검증부터 결과를 확인합니다.>
5. <결과, 미실행 검증과 남은 위험을 보고합니다.>

<필요한 경우에만 조건 분기와 중단 기준을 추가합니다.>

## 검증 기준

- <실행하거나 검토할 검사>
- <성공으로 판단할 기대 결과>
- <실패 시 중단, 수정 또는 재확인이 필요한 조건>

## 금지 패턴

- <범위 밖 변경 또는 근거 없는 추정>
- <검증 생략, 위험한 명령 또는 secret 노출>
- <이 workflow에서 특별히 금지할 행동>

## 완료 기준

- <요청 결과가 충족되었음을 보여 주는 산출물>
- <통과해야 하는 검증과 보존해야 하는 기존 동작>
- <실행하지 못한 검증과 남은 위험이 명시됨>

## 참고 문서

- <프로젝트 내부의 기준 문서 또는 공식 문서 경로>
- <참고 문서가 없으면 `없음`>
