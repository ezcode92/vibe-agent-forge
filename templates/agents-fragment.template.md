# <Fragment 표시 이름>

> 이 파일은 stack fragment의 구조를 정의하는 템플릿입니다. `<...>` 표시는 작성 시 실제 값으로 교체하고, 해당 stack에 고유하지 않은 공통 규칙은 포함하지 않습니다.

## 목적

<이 fragment가 담당하는 기술 영역과 해결하려는 문제를 설명합니다. 다른 fragment나 skill이 담당할 범위도 함께 구분합니다.>

## 적용 대상

- Stack 식별자: `<registry-stack-id>`
- 적용 조건: <이 fragment를 선택해야 하는 조건>
- 제외 조건: <적용하지 않아야 하는 조건>

## 핵심 규칙

- <항상 지켜야 하는 명확하고 검증 가능한 규칙>
- <이 stack에서만 달라지는 책임 또는 제약>

## 권장 패턴

- <문제를 단순하게 해결하는 관용적 패턴과 선택 기준>
- <상황에 따라 적용할 패턴과 적용 조건>

## 금지 패턴

- <회귀, 결합도 증가 또는 생태계 관례 위반을 일으키는 패턴>
- <근거 없이 사용하면 안 되는 도구나 구조>

## 검증 기준

- <규칙 준수 여부를 확인할 수 있는 정적 검사 또는 리뷰 기준>
- <실행 가능한 경우 사용할 테스트·lint·build 범주와 기대 결과>
- <검증할 수 없을 때 보고해야 할 제한 사항>

## 관련 skill

- 필수 skill: <없으면 `없음`, 있으면 registry skill 식별자와 필요한 이유>
- 선택 skill: <특정 작업 조건에서만 사용하는 registry skill 식별자와 조건>

Skill의 작업 절차를 이 문서에 복제하지 않습니다.

## Bridge 필요 조건

- <다른 stack과 함께 선택될 때만 유효한 연결 규칙의 발생 조건>
- 필요한 bridge: <없으면 `없음`, 있으면 registry bridge 식별자>

둘 이상의 stack 사이에서만 필요한 interface 또는 lifecycle 규칙은 이 fragment에 넣지 않고 bridge fragment로 분리합니다.
